## 中文

感谢您考虑为 Enhanced Download Manager 做出贡献！

### 如何贡献

1. **Fork 本仓库**
2. **创建您的特性分支**
   ```bash
   git checkout -b feature/AmazingFeature
   ```
3. **提交您的更改**
   ```bash
   git commit -m 'Add some AmazingFeature'
   ```
4. **推送到分支**
   ```bash
   git push origin feature/AmazingFeature
   ```
5. **开启 Pull Request**

### 代码规范

- 遵循 [WordPress PHP 编码标准](https://developer.wordpress.org/coding-standards/wordpress-coding-standards/php/)
- 使用有意义的变量和函数名
- 添加必要的注释
- 确保所有输出都经过适当的转义
- 验证所有用户输入

### 提交信息规范

使用清晰的提交信息：

- `feat: 添加新功能`
- `fix: 修复bug`
- `docs: 更新文档`
- `style: 代码格式调整`
- `refactor: 重构代码`
- `perf: 性能优化`
- `test: 添加测试`

### 报告 Bug

在提交 Bug 报告时，请包含：

- WordPress 版本
- PHP 版本
- 插件版本
- 问题描述
- 重现步骤
- 预期行为
- 实际行为
- 截图（如果适用）

### 功能请求

我们欢迎功能请求！请：

1. 检查是否已有类似的请求
2. 清楚地描述您想要的功能
3. 说明为什么这个功能有用
4. 提供使用场景示例

## English

Thank you for considering contributing to Enhanced Download Manager!

### How to Contribute

1. **Fork the repository**
2. **Create your feature branch**
   ```bash
   git checkout -b feature/AmazingFeature
   ```
3. **Commit your changes**
   ```bash
   git commit -m 'Add some AmazingFeature'
   ```
4. **Push to the branch**
   ```bash
   git push origin feature/AmazingFeature
   ```
5. **Open a Pull Request**

### Code Standards

- Follow [WordPress PHP Coding Standards](https://developer.wordpress.org/coding-standards/wordpress-coding-standards/php/)
- Use meaningful variable and function names
- Add necessary comments
- Ensure all outputs are properly escaped
- Validate all user inputs

### Commit Message Guidelines

Use clear commit messages:

- `feat: Add new feature`
- `fix: Fix bug`
- `docs: Update documentation`
- `style: Code formatting`
- `refactor: Refactor code`
- `perf: Performance optimization`
- `test: Add tests`

### Bug Reports

When submitting a bug report, please include:

- WordPress version
- PHP version
- Plugin version
- Description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)

### Feature Requests

We welcome feature requests! Please:

1. Check if a similar request already exists
2. Clearly describe the feature you want
3. Explain why this feature would be useful
4. Provide use case examples
